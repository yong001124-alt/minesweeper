## 二、主题叙事框架（v2 · 猫咪第一人称）

### 世界观设定
> 我是一只独自生活的小猫。铲屎的不在，没有人帮我清理猫砂盆。  
> 每次想上厕所，都成了一场冒险——  
> 我得靠鼻子（数字线索）嗅出哪里埋着我以前拉的💩，  
> 踮起爪子，找到一块干净的砂地，优雅地完成这次如厕。  
> **绝对、绝对不能踩到自己拉过的地方！**（猫的尊严不允许）

### 核心元素映射

| 原版扫雷 | 猫咪版本 | 说明 |
|----------|-----------|------|
| 格子（未翻开） | 未探明的砂堆 🟫 | 不知道下面埋了什么，圆润拟物风 |
| 翻开格（安全） | 探明的干净砂地 ✨ | 爪子确认过安全的落脚点，浅米色 |
| 地雷 💣 | 我以前拉的便便 💩 | 自己埋下的「地雷」，圆润棕色小堆 |
| 数字1-8 | 鼻子嗅出的线索 👃 | 「这格周围埋着 N 坨」，颜色体系保留 |
| 旗帜 🚩 | 猫咪记号 🐾 | 爪印标记：「这里有屎，我记住了」 |
| 踩雷失败 | 踩到自己的便便 😱 | 猫洁癖崩溃 + 棕色粒子飞溅 |
| 通关成功 | 优雅如厕成功 😸 | 心满意足刨砂掩埋 + 爱心粒子 |

## 三、视觉风格指南

### 3.1 整体基调

- **关键词**：毛茸茸 / 圆润 / 温暖 / 可爱 / 明快
- **风格参考**：日系暖萌风（类似 Neko Atsume、LINE Friends）
- **基准参考**：Neko Station（Steam，截图存于项目根目录 `ss_*.jpg`）——16-bit 像素艺术 · 低饱和高温度 · 治愈系
- **禁忌**：不使用冷色系硬朗设计、不使用恐怖/紧张气氛

### 3.2 配色系统

```
背景/场景色（Neko Station 天空+草地色调）
  --sky-blue:    #A8D4E6  ← 天空蓝（背景主色）
  --cloud-white: #F0EDE4  ← 云朵奶白（卡片/弹窗背景）
  --grass-green: #8DC87A  ← 草地绿（通关/安全色）

格子色（猫砂盆色调）
  --sand-undig:  #C9A96E  ← 未铲开：暖沙棕（渐变到 #B8935A）
  --sand-dug:    #EDE0C4  ← 铲开后：奶白沙色
  --sand-border: #A07848  ← 格子边框：深棕（立体感）

强调色（车厢内装饰色调）
  --sakura-pink: #F2A7BB  ← 樱花粉（主按钮/强调）
  --wood-brown:  #8B6355  ← 原木棕（文字/次要边框）
  --paw-orange:  #E8855A  ← 猫爪橙（当前关卡/激活态）

功能色
  --poop-brown:  #6B4226  ← 便便棕（cell.mine 揭示色）
  --flag-red:    #E05A5A  ← 标记红（旗帜/警示）
  --safe-green:  #8DC87A  ← 安全绿（通关/里程碑）
```

### 3.3 字体

```
标题/UI字体：系统圆体（保证移动端可读性）
  推荐：system-ui / "PingFang SC" / "Noto Sans SC"
  特点：字重适中，圆润感，不用细体，不用像素字体

棋盘内数字：
  等宽圆体，颜色走暖色像素系（见 CLAUDE.md 数字色表）
  1=猫爪橙 2=草地绿 3=标记红 4=沙棕 5=樱花粉 6=天空蓝 7=奶白 8=深棕

正文/UI文案：
  14-16px，行高 1.6，原木棕 #8B6355
```

### 3.4 格子视觉规格

```
未探明格（砂堆）—— 像素立体风格：
  ├ 背景：#C9A96E 渐变至 #B8935A（左上→右下）
  ├ 上边/左边高光：1px #DDBF82（亮边，模拟像素光源左上角）
  ├ 下边/右边阴影：1px #8A6030（暗边，模拟像素立体）
  ├ 内部：轻微点噪声模拟砂粒颗粒感
  └ 圆角：2px（像素风保持方正，略微软化）

探明格（干净砂地）：
  ├ 背景色：#EDE0C4（微内凹感，inset shadow）
  ├ 边框：1px inset #C8B48A
  └ 动画：探开时有短暂「砂子飞溅」粒子（3-5颗小点）

便便格（被发现）：
  ├ 中心：像素风圆润💩（CSS绘制或 emoji）
  ├ 背景：浅暖红晕 #FFE8D6
  └ 动画：棕色像素粒子爆开 + 臭气线向上飘

记号格（已标记）：
  ├ 图标：像素猫爪 🐾
  └ 颜色：猫爪橙 #FF8C42
```

> **场景换肤**：以上是「猫砂盆」基准皮肤。进入小房间/公寓/别墅场景后，
> 未探明格纹理与棋盘外框随场景变化（见第五节），格子机制与尺寸规则不变。

## 四、关卡地图（v2 · 封闭上升路线）

### 4.1 设计原则

1. **封闭路线**：一条单一的上升路径，所有关卡串在同一条线上，没有岔路。
2. **严格顺序解锁**：必须通关第 N 关才能解锁第 N+1 关；未解锁节点不可点击，**没有任何跳关入口**（教学关除外，可跳过教学直接进第 1 关）。
3. **竖屏优先**：信息架构按手机竖屏单手操作习惯设计。

### 4.2 竖屏布局规范

```
┌─────────────────────────────┐
│  😺 我的如厕等级：踮脚小猫 Lv.2 │ ← 顶部状态栏（sticky，含勋章入口）
├─────────────────────────────┤
│         🏰 别墅              │
│      [20]                   │    ↑ 路径自下而上蜿蜒攀升
│         [19]                │      （越往上＝越高级的场景）
│      [18]   …               │
│  ═══ 🏢 公寓（13–16）═══      │ ← 章节分隔横幅（场景名+背景换色）
│         [13🔒]              │
│  ═══ 🚪 小房间（9–12）═══     │
│      [12🔒]                 │
│         [11🔒]              │
│      [10🔒]                 │
│         [9🔒]               │
│  ═══ 🛁 大号猫砂盆（5–8）═══   │
│      [8 ▶]  ← 当前关卡       │
│         [7✓]                │
│      [6✓]                   │
│         [5✓]                │
│  ═══ 🥣 小号猫砂盆（1–4）═══   │
│      [4✓] [3✓] [2✓] [1✓]    │
└─────────────────────────────┘
```

- **滚动方向**：列表底部=第1关，顶部=第20关；进入页面时自动滚动到「当前关卡」，使其位于屏幕中部偏下（拇指热区），向上可预览后续关卡。
- **节点规格**：圆形节点直径 ≥ 56px（≥44px 最小触控目标 + 余量），节点间距 ≥ 16px。
- **单列蜿蜒**：节点在中轴线左右交替摆动（振幅约屏宽 18%），SVG 爪印虚线连接（已完成段绿色）。
- **章节横幅**：迷你勋章图标 + 场景名 + 关卡范围，随场景切换背景色调；横幅吸顶（sticky）提示当前所在场景。
- **整幅场景背景（已与用户确认，禁用 Emoji 点缀）**：每个章节的背景为**铺满整段的场景材质 + 手绘像素/CSS 道具**——砂地（颗粒纹理+砂丘+猫砂盆+爪印）→ 浴室格砖（带罩猫厕）→ 房间墙纸（窗户+地毯+毛线球）→ 公寓墙面（亮灯窗排+盆栽）→ 别墅大理石（树木+树篱+闪光）；中轴保留浅色通道保证路径与节点可读；顶部为 Key Art 同款天空（太阳/云朵/双层草丘），山顶终点为**像素别墅**（非 Emoji）。
- **勋章同步（已与用户确认）**：章节横幅图标与等级勋章使用同一套素材；各场景末关（4/8/12/16/20）节点挂「通关即颁发」的勋章角标，未达成时显示灰色剪影。

### 4.3 关卡节点样式

```
未解锁关卡：
  ├ 灰色沙色 #D4C4A0
  ├ 🔒 锁图标
  └ 微微透明，点击无效（轻微抖动反馈「还没到这里哦」）

当前关卡：
  ├ 亮橙 #FF8C42 + 白边
  ├ 轻微弹跳动画（bounce）
  └ 猫爪图标 ▶

已通关：
  ├ 暖绿 #A8D8A8
  ├ ✓ 勾选标记
  └ 获得星级（1-3星）显示在节点下方

关卡连接线：
  ├ 爪印点线（猫爪印连成路径）
  └ 已完成段：暖绿色；未完成段：灰色
```

## 五、棋盘场景升级体系（v2 新增）

随等级上升，如厕冒险的「战场」不断扩大——从一只小猫砂盆，一路升级到整栋别墅（毕竟没人收拾，哪里都可能有💩）。

| 场景 | 关卡 | 棋盘规模趋势 | 视觉主题 | 叙事 |
|------|------|------------|---------|------|
| 🥣 小号猫砂盆 | 1–4 | 5×5 → 8×8 | 塑料盆圆角外框、米色细砂 | 我的第一个猫砂盆 |
| 🛁 大号猫砂盆 | 5–8 | 9×9 → 11×14 | 加大盆、双色砂、盆沿高光 | 升级了大豪盆！ |
| 🚪 小房间 | 9–12 | 12×16 → 13×20 | 木地板纹理、踢脚线外框、散落砂粒 | 砂盆满了…只能去房间角落 |
| 🏢 公寓 | 13–16 | 14×22 → 15×26 | 多色地砖、地毯块、家具剪影点缀 | 整个公寓都是我的厕所 |
| 🏰 别墅 | 17–20 | 16×28 → 16×30 | 大理石纹、花园草地边框、吊灯光斑 | 别墅如厕，至高荣耀 |

**实现要点**：
- 场景只换「皮肤」：棋盘外框装饰、未探明格纹理/色调、页面背景；格子尺寸计算、扫雷逻辑、触控逻辑完全复用。
- 每个场景一组 CSS 变量（边框色/格子渐变/背景渐变），由 `levelIdx` 推导场景索引切换。
- 关卡数值表（rows/cols/mines）保持现有 20 关瀑布式难度曲线，按上表范围对齐。

## 六、启动流程（v2 新增）

### 6.1 首次进入：故事背景动画（phase = "intro"）

以 `localStorage.seenIntro` 判断是否首次。动画用 CSS 关键帧 + Emoji/简笔图形实现（不引入外部资源），共 4 幕，每幕约 3s，自动播放，右上角「跳过」常驻。

**风格约束（已与用户确认）**：与主视觉 Key Art 元素完全一致——字幕用同款木质招牌（带钉子）；窗户/地面材质/星月/角色投影均复用 Key Art 素材；第1幕夜景（月亮+星星+木地板）、第2/3幕室内（窗外有云+地板）、第4幕白天砂地（太阳云朵+砂粒地面+爪印足迹）。

| 幕 | 画面 | 旁白文案（第一人称） |
|----|------|---------------------|
| 1 | 小猫独坐窗边，房间空荡 | 「铲屎的出远门了。这个家，现在归我管。」 |
| 2 | 猫砂盆特写，上方冒出 💩💩💩 | 「但是……没有人铲屎了。」 |
| 3 | 小猫憋红了脸，腿夹紧 | 「可我还是得上厕所啊！！」 |

## 七、用户成长体系（v2 新增 · 等级与勋章）

### 7.1 等级体系（与场景同步）

玩家等级随通关进度提升，等级与棋盘场景一一对应（通过场景最后一关即升级）：

勋章设计原则（已与用户确认）：
1. **猫咪是主角**——称号一律是猫自己（「XX小猫」），禁用「达人」等类人类称谓；
2. **连贯叙事线**——六枚勋章 = 猫咪如厕仪式的完整六步（踏砂→嗅嗅→踮爪→刨坑→埋宝→加冕），且与玩法一一呼应；
3. **禁用禁止符**类符号设计；
4. **16×16 多色像素**（奶油/粉鼻/砂色/白瓷分色），细节充足、一眼可读，不用大色块。

| 等级 | 称号 | 解锁条件 | 勋章图案 | 玩法呼应 |
|------|------|---------|---------|---------|
| Lv.1 | 踏砂小猫 | 初始 | 肉垫爪踏进猫砂盆（绿底） | 第一次进入游戏 |
| Lv.2 | 嗅嗅小猫 | 通关第 4 关 | 大粉鼻猫脸+金色气味波（蓝底） | 数字=鼻子嗅出的线索 |
| Lv.3 | 踮爪小猫 | 通关第 8 关 | 单爪踮在砂尖+轻盈星点（粉底） | 踮脚避开便便 |
| Lv.4 | 刨坑小猫 | 通关第 12 关 | 爪刨砂坑+砂粒飞溅（蓝灰底） | 探砂动作 |
| Lv.5 | 埋宝小猫 | 通关第 16 关 | 砂丘掩埋只露便便尖+闪光（深棕底） | 通关掩埋仪式 |
| 终极 | 如厕之王 | 通关第 20 关 | 戴皇冠的猫坐白瓷马桶王座（金渐变底+闪光动效） | 全部通关加冕 |

### 7.2 勋章设计规范（对齐 Neko Station 像素风基准）

- **风格**：16-bit 像素徽章，圆形底盘（直径 64px 网格），低饱和高温度配色（用 3.2 配色系统取色），1px 深色描边 + 左上高光，与项目最初的 Neko Station 视觉基准一致。
- **绘制方式**：CSS box-shadow 像素画或内联 SVG（不引入图片资源、不加 npm 包）。
- **素材复用**：勋章图标与关卡地图共用同一套素材——章节横幅前缀迷你勋章、各场景末关节点挂「通关即颁发」勋章角标（未达成时灰色剪影）。
- **颁发时机**：升级瞬间（通关该场景最后一关的 Splash 之后）弹出「升级仪式」全屏卡片：勋章从中心放大入场 + 爪印粒子环绕 + 「喵～」音效变奏。
- **勋章墙**：从首页/关卡地图顶栏进入；按获得顺序排列，未获得的显示剪影 + 解锁条件；数据存 `localStorage`。

## 八、游戏内 UI

### 8.1 顶部状态栏（HUD）

```
[😺 还埋着3坨💩]          [⏱ 02:34]          [🐾 第8关]
```

- `💩` 计数器（剩余未标记的便便数）
- 猫咪表情随游戏状态变化：😺 正常 → 😱 踩到了 → 😸 如厕成功

### 8.2 底部操作栏

```
┌──────────────────────┬──────────┬──────────┐
│ [🐾 探砂模式] [🚩 记号] │  😺/😱/😸  │   🏠     │
│  ← 滑块切换 →        │  重试按钮  │  返回首页 │
└──────────────────────┴──────────┴──────────┘
```

- 探砂模式 = 原挖掘模式（用爪子确认这里安全）
- 记号模式 = 原插旗模式（标记「这里有屎」）

### 8.3 失败覆盖层

```
  😱 踩到自己拉的了！！

  [便便飞溅动画]  ← 棕色粒子 + 臭气线

  点击任意位置再试一次
```

### 8.4 通关弹窗（Splash）

### 10.3 猫咪叫声体系（v2 新增需求）

第一人称叙事下，猫咪的「声音」就是玩家自己的情绪反馈。在现有合成音效基础上，
增加一组**喵声变体**（全部 Web Audio API 合成，参考现有 playLevelUp 的三角波 meow 手法：
基频滑音 + vibrato + 短促包络，靠音高曲线区分情绪）：

| 时机 | 叫声 | 情绪/音色描述 |
|------|------|--------------|
| 进入关卡 | 元气短喵「喵!」 | 短促上扬，基频 600→900Hz，约 0.25s |
| 大片连开（cascade） | 兴奋喵「喵喵～」 | 双音节，第二声更高，轻快 |
| 里程碑 50%/75% | 满足喵（叠在呼噜声上） | 低缓下行，慵懒 |
| 踩到便便 | 哀怨长喵「喵呜……」 | 下行长音 + 颤音，约 0.8s（叠在现有惊叫之后） |
| 通关 | 「喵～」（已有 ✅） | 保持现状 |
| 升级颁章 | 喵声变奏 + 钟声泛音 | 庄重版长喵 |
| 重试/再来一次 | 振作短哼「嗯喵」 | 鼻音短促，重新开始的干劲 |

- 音量层级：叫声 ≤ 操作音效，不抢主反馈；同一时刻只播一个喵声（后到打断先到）。
- 所有叫声共用一个 `playMeow(type)` 合成函数，按类型查表取音高曲线/时长参数。
